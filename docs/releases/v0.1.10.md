Changes since v0.1.9.

The desktop release: the viewer leaves TCP for a unix socket, a tray-resident macOS app wraps it,
and the whole stack ships through Homebrew with a signed, notarized universal app bundle. The
knowledge model and the MCP tool surface are unchanged.

## Highlights

### BREAKING - the viewer serves over a unix socket, never TCP
The viewer's localhost port invited the browser attack class the last two releases kept patching
(DNS rebinding, CSRF, cross-site fetch). It now binds a 0600 unix socket (default
`~/.supragnosis/viz.sock`); the socket file is the entire access control, and the Host/CSRF gates
are deleted rather than defended. `SUPRAGNOSIS_VIZ_ADDR` and `SUPRAGNOSIS_VIZ_PUBLIC` are gone -
`SUPRAGNOSIS_VIZ_SOCK` / `--viz <path>` replace them, and a stale variable warns loudly at startup.
Reads still work from any HTTP-over-UDS client (`curl --unix-socket`); the MCP http daemon keeps
its loopback TCP bind until the stdio proxy shim lands. `serve --http` now brings the viewer
socket up by default, exactly like `start`, so supervisor-run daemons (brew services, systemd)
serve the human channel too.

### A macOS desktop app
`app/` is a tray-resident Tauri shell over the daemon-served viewer - it embeds no frontend and no
engine (the daemon owns the db; one UI source serves desktop today and the hub read tier later).
The shell attaches to a running daemon's socket or spawns one and reaps it on quit, proxies the
webview through a `viz://` custom protocol onto the unix socket, bridges the SSE event stream, and
lives in the menu bar: closing the window keeps the daemon running, the tray reopens the viewer,
shows daemon status, restarts it, and quits. The macOS title bar merges into the viewer header
(overlay traffic lights, header drag region, fullscreen-aware padding) - all injected by the
shell, leaving the shared viewer assets browser-neutral.

### Homebrew distribution, signed and notarized
`brew tap ashon/tap` then `brew install supragnosis` (daemon/CLI formula with a `brew services`
block) and `brew install --cask supragnosis-app` (the desktop app). Releases attach a
Developer-ID-signed, notarized, stapled universal `.app` zip built by CI - Gatekeeper accepts it
as `Notarized Developer ID`, no overrides needed. The cask depends on the formula: the app uses
the brew-managed daemon binary, so no sidecar is bundled and `brew upgrade` moves both together.

### Viewer UX pass
Per-dock drawer handles ride the panel edge and slide with it (`[` / `]` shortcuts, state
persisted); view options move off the data rails into a compact checkbox popover behind the HUD's
eye button; the HUD renders inline stroke SVGs (CDN-free); every floating surface shares one glass
material; and the app icon, tray mark, and favicons all draw the same asterisk geometry.
