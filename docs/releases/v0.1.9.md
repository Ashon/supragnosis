Changes since v0.1.8.

A viewer maintainability release: the ontology viewer's frontend moves out of an inline Rust string
into real files, gains a security lint, and is minified at build time. No knowledge-model, wire, or
API changes - the graph engine and every HTTP surface behave as before.

## Highlights

### The viewer frontend is real files now
The viewer was a single ~1900-line inline HTML/CSS/JS raw string inside `lib.rs`. It now lives in
`crates/supragnosis-viz/assets/` as `viewer.html` / `viewer.css` / `viewer.js`, embedded at compile
time via `include_str!` and served as separate same-origin routes (`/`, `/viewer.css`, `/viewer.js`).
The crate is still a single self-contained binary that works offline (no CDN), and `lib.rs` drops
from ~2600 to ~660 lines (just the HTTP server).

### An ESLint guard on the innerHTML sinks
Splitting the script out lets a linter see it. `assets/` carries an `eslint-plugin-no-unsanitized`
config that flags every `innerHTML` / `insertAdjacentHTML` / `document.write` sink with non-literal
content - the XSS class (Principle 18) that previously hid unnoticed in the inline string. Each vetted
sink carries an explicit "value is built from esc()-escaped strings" disable directive, so the HTML
sinks are greppable and any new one fails the lint until it is escaped and consciously acknowledged.
A `frontend-lint` CI job runs it on changes under `assets/`. The crate build does not use it - the
build stays pure cargo (Node is only for the lint).

### Build-time minification (release only)
A `build.rs` minifies the assets into `OUT_DIR`, release builds only, so debug still serves the
readable files. Pure cargo - the minifiers are Rust build-dependencies, not a Node toolchain:

- CSS via `lightningcss` (-30%)
- HTML via `minify-html` (-10%)
- JS via the `oxc` code generator in minify mode with comments off: whitespace and comments removed,
  but **no identifier mangling and no dead-code elimination** (semantics-preserving printing) -
  `viewer.js` roughly -50% (~99KB -> ~50KB). Parsing also fails the build on malformed JS, a free
  correctness check.
